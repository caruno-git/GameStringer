## 概要

GameStringerはビデオゲームの自動・手動翻訳のための高度なシステムです。以下をサポート:

- **ゲームエンジン**: Unity、Unreal Engine、RPG Maker、Ren'Py、Godot、Telltale、Wolf RPG、Kirikiriなど
- **ファイル形式**: CSV、JSON、XML、PO/POT、YAML、TXT、SRT、VTT、ASS/SSAなど
- **AIプロバイダー**: Claude、Gemini、GPT、DeepSeek、Mistral、Groq、Ollama、**Qwen 3**、**NLLB-200**（18以上のプロバイダー）
- **言語**: 200以上の対応言語（NLLB-200使用時）
- **多言語UI**: IT、EN、ES、FR、DE、JA、ZH
- **ゲームストア**: Steam、Epic Games、GOG、Origin、Battle.net、Ubisoft、itch.io、Amazon Games
- **新機能 v1.0.5**: Voice Clone Studio、VR Text Overlay、Quality Gates、Player Feedback
- **新機能 v1.0.6**: Qwen 3（アジア言語）、NLLB-200（200言語）、バグ修正
- **新機能 v1.0.7**: Community Hub、GitHub Discussions、ライセンス v1.1
- **新機能 v1.0.8**: アップデートダウンロード修正
- **新機能 v1.0.9**: アニメーションヘッダー、アップデート通知、UIポリッシュ

## 初回起動とプロファイル

### プロファイル作成

初回起動時、GameStringerはユーザープロファイルの作成を求めます：

1. **「プロファイル作成」をクリック**（初期画面）
2. **プロファイル名を入力**（例：「私の名前」）
3. **パスワードを設定**（最低6文字）
4. **「作成」をクリック**して確認

### ログイン

既存のプロファイルにアクセスするには：

1. **リストからプロファイルを選択**
2. **パスワードを入力**
3. **（オプション）**「パスワードを記憶」にチェックで自動ログイン
4. **「ログイン」をクリック**

### プロファイル管理

- **プロファイル切替**: 右上のプロファイルアイコンをクリック → 「プロファイル切替」
- **ログアウト**: プロファイルアイコンをクリック → 「ログアウト」
- **プロファイル設定**: 設定 → プロファイルへ

## ライブラリとゲーム詳細

### ライブラリ

ライブラリにはSteam、Epic Games、GOGなどのストアから同期された全ゲームが表示されます。

- **更新**: ゲームリストを再読み込み
- **共有**: ファミリーシェアリングゲームの表示/非表示
- **フィルター**: プラットフォーム、インストール状態、エンジンでフィルタリング

### ゲーム詳細ページ

ゲームをクリックすると**3:1**レイアウトの詳細ページが開きます：

#### メインカラム（75%）

- **スクリーンショットギャラリー**: クリック可能な最大12枚のスクリーンショット（ライトボックス）
- **クイック情報**: エンジン、ファイル数、インストールパス、DLC
- **ファイル/翻訳/パッチタブ**:
  - **ファイル**: 「Neural Translator」ボタン付きの翻訳可能ファイル
  - **翻訳**: このゲームのアクティブな翻訳
  - **パッチ**: Unity、Unreal、RPG Maker用パッチのインストール/削除

#### 右サイドバー（25%）

- **ゲーム情報**: 開発者、パブリッシャー、発売日、ジャンル、対応言語
- **アクション**: ゲームを翻訳、ファイルをスキャン
- **HowLongToBeat**: ゲームクリアの推定時間

#### 翻訳推奨

ページ下部でシステムがゲームを分析し、**最適な翻訳方法**を提案：

| 方法 | 使用タイミング |
|------|---------------|
| **Live Unity** | BepInEx + XUnity搭載のUnityゲーム |
| **File Translation** | ローカライズファイルが見つかった場合（JSON、CSVなど） |
| **OCR Overlay** | ファイルが見つからない場合、リアルタイム視覚翻訳 |

## Neural Translator Pro

### ファイルの翻訳方法

1. **Steamライブラリからゲームを選択**または手動でアップロード
2. **翻訳するファイルを読み込み**（ドラッグ＆ドロップまたは参照）
3. **オプションを設定**:
   - **ソース言語**: 元の言語（例：英語）
   - **ターゲット言語**: 翻訳先の言語（例：日本語）
   - **AIプロバイダー**: Claude（推奨）、Gemini、GPT
   - **APIキー**: 選択したプロバイダーのAPIキーを入力
4. **「翻訳開始」をクリック**して翻訳を開始
5. **進捗バーで進行状況を確認**

### 詳細オプション

| オプション | 説明 |
|----------|------|
| **Quality Checks** | 自動品質チェック（数字、フォーマットなど） |
| **Translation Memory** | 以前の翻訳を再利用して高速化 |
| **Batch Size** | 並列翻訳する文字列数（デフォルト: 10） |

### 推定コスト

システムは開始前にコスト見積もりを表示：
- **Claude**: 1Kトークンあたり〜$0.003
- **Gemini**: 1Kトークンあたり〜$0.0005（より安価）
- **GPT-4**: 1Kトークンあたり〜$0.01

## Translation Wizard

Translation Wizardはゲームファイルを自動翻訳するためのガイド付き手順です。

### Wizardの使い方

1. **Translatorに移動** → 「Translation Wizard」をクリック
2. **ライブラリからゲームを選択**またはパスを手動入力
3. **ファイルをスキャン**: Wizardが自動的に翻訳可能ファイルを検出
4. **翻訳するファイルを選択**（複数選択可能）
5. **オプションを設定**:
   - ソース言語とターゲット言語
   - AIプロバイダー
   - 品質オプション
6. **バッチ翻訳を開始**
7. **進捗バーで進行状況を確認**

### 自動検出フォーマット

| 拡張子 | タイプ |
|--------|--------|
| `.json` | JSONローカライズ |
| `.csv` | テキストテーブル |
| `.xml` | XML設定 |
| `.po/.pot` | Gettext（Linux標準） |
| `.txt` | プレーンテキスト |
| `.yaml` | YAML設定 |

## Translation Bridge

Translation BridgeはUnityゲームを**リアルタイム**でプレイ中に翻訳できます。

### 要件

- Unityゲーム（MonoまたはIL2CPP）
- BepInExインストール済み
- XUnity.AutoTranslatorプラグイン

### 設定方法

1. **メニューからTranslation Bridgeに移動**
2. **リストからUnityゲームを選択**
3. **BepInExをインストール**（未インストールの場合は自動）
4. **XUnity.AutoTranslatorを設定**:
   - ターゲット言語
   - 翻訳エンドポイント
5. **ゲームを起動** - 翻訳が自動的に表示されます

### 動作モード

- **ローカルキャッシュ**: 翻訳を再利用のために保存
- **ライブ翻訳**: 新しい文字列をその場で翻訳
- **フォールバック**: オフライン時はキャッシュのみ使用

## 公開API v1

*(v1.0.4の新機能)*

GameStringerは外部統合用のREST APIを提供します。

### 利用可能なエンドポイント

| メソッド | エンドポイント | 説明 |
|---------|---------------|------|
| `POST` | `/api/v1/translate` | 単一文字列の翻訳 |
| `POST` | `/api/v1/batch` | バッチ翻訳（最大100文字列） |
| `GET` | `/api/v1/languages` | 対応20言語のリスト |
| `GET` | `/api/v1/health` | サービスヘルスチェック |

### リクエスト例

```json
POST /api/v1/translate
{
  "text": "Hello, world!",
  "source": "en",
  "target": "ja",
  "provider": "gemini"
}
```

### レスポンス例

```json
{
  "translation": "こんにちは、世界！",
  "source": "en",
  "target": "ja",
  "provider": "gemini",
  "tokens": 12
}
```

### CI/CD利用

APIは自動ビルドパイプラインへのGameStringer統合に最適です。

## Quality Gates

*(v1.0.5の新機能)*

翻訳の自動品質管理システム。

### 自動チェック

| チェック | 説明 |
|---------|------|
| **Placeholder** | {0}、{1}、%sなどを検証 |
| **数字** | 数字が正しく保持されているか |
| **HTMLタグ** | `<color>`、`<b>`などが無傷か |
| **長さ** | 翻訳が長すぎ/短すぎないか |
| **句読点** | 元のテキストとの一貫性 |

### 信頼度レベル

| レベル | スコア | 色 |
|--------|--------|-----|
| 🔴 クリティカル | < 40% | 赤 |
| 🟠 低 | 40-59% | オレンジ |
| 🟡 中 | 60-74% | 黄 |
| 🟢 高 | 75-89% | 緑 |
| 💚 完璧 | 90-100% | 濃緑 |

### 使い方

1. **メニューからQuality Gatesに移動**
2. **翻訳を読み込み**（JSON、CSV、または貼り付け）
3. **各文字列を自動分析**
4. **信頼度レベルでフィルタリング**
5. **JSONでレポートをエクスポート**

## パッチエクスポート

Unity PatcherはUnityゲームにBepInExとXUnity.AutoTranslatorを自動インストールします。

### 使い方

1. **サイドバーからUnity Patcherに移動**
2. **リストからUnityゲームを選択**（緑の「Unity」バッジ）
3. **ターゲット言語を選択**（例：日本語）
4. **モードを選択**:
   - **キャプチャのみ**: 手動翻訳用にテキストをキャプチャ
   - **Google翻訳**: ゲーム内自動翻訳
   - **DeepL**: より高品質な自動翻訳
5. **「パッチをインストール」をクリック**
6. **ゲームを起動** - `ALT+T`でXUnityメニューを開く

### 翻訳バッジ

インストール後、状態を示すバッジが表示されます:

| バッジ | 意味 |
|--------|------|
| 🥈 シルバー | 自動翻訳有効なXUnity（Google/DeepL） |
| 🥉 ブロンズ | テキストキャプチャのみ（手動翻訳） |

### アクティビティトラッキング

インストールされた各パッチはダッシュボードの**最近のアクティビティ**に記録:
- ゲーム名
- 選択した翻訳モード
- ターゲット言語

## アクティビティ履歴

アクティビティ履歴は実行されたすべての操作を追跡します。

### アクセス

サイドバーから**アクティビティ履歴**に移動。

### 記録されるアクティビティタイプ

| アイコン | タイプ | 説明 |
|---------|--------|------|
| 🌐 | Translation | 完了した翻訳 |
| 🔧 | Patch | 作成/適用されたパッチ |
| ♻️ | SteamSync | Steam同期 |
| ➕ | GameAdded | 追加されたゲーム |
| 🎮 | GameLaunched | 起動されたゲーム |
| 👤 | ProfileCreated | 作成されたプロファイル |
| ⚙️ | SettingsChanged | 変更された設定 |

### 利用可能なフィルター

- **タイプ別**: アクティビティカテゴリでフィルタリング
- **日付別**: 時間範囲を選択
- **ゲーム別**: 特定ゲームのアクティビティのみ表示

## パッチエクスポート

翻訳完了後、配布用パッケージをエクスポートできます。

### 「パッチをエクスポート」ボタン

**デスクトップ**に以下を含むZIPファイルを作成:

```
📦 ゲーム名_ja_patch.zip
├── 📁 translated/          # 使用可能な翻訳済みファイル
│   └── 翻訳済みファイル.csv
├── 📁 backup/               # 元ファイルのバックアップ
│   └── 元ファイル.csv
├── 📁 xunity/               # XUnity.AutoTranslator形式
│   └── AutoTranslator/
│       └── Translation/
│           └── ja/
│               └── _Translations.txt
├── 📄 README.txt            # インストール手順
└── 📄 metadata.json         # 翻訳情報
```

### XUnity.AutoTranslator形式

XUnity形式は以下と互換:
- BepInEx + XUnity.AutoTranslator搭載の**Unityゲーム**
- 形式: `元テキスト=翻訳テキスト`

## ゲームに適用

### 「ゲームに適用」ボタン

翻訳を**ゲームに直接**自動インストール:

1. **ゲームのエンジンを検出**（Unity、Unrealなど）
2. **利用可能なパッチャーとの互換性を確認**
3. **必要に応じてパッチャーをインストール**（例：Unity用BepInEx）
4. **翻訳ファイルを正しいフォルダにコピー**
5. **翻訳を読み込むようゲームを設定**

### 対応エンジン

| エンジン | パッチャー | 状態 |
|---------|-----------|------|
| Unity (Mono) | BepInEx + XUnity.AutoTranslator | ✅ 自動 |
| Unity (IL2CPP) | BepInEx IL2CPP | ⚠️ 部分的 |
| Unreal Engine | - | 🔧 手動 |
| RPG Maker | - | ✅ 直接置換 |
| Ren'Py | - | ✅ 直接置換 |

### 元ファイルはどうなる？

**元ファイルは常に保持されます！**

1. 上書き前に自動バックアップを作成
2. バックアップはゲームフォルダの`.gamestringer_backups/`に保存
3. バックアップ名にはタイムスタンプを含む: `20241228_001500_ファイル名.csv`

## バックアップ管理

### バックアップの場所

バックアップは2か所に保存:

1. **ゲームフォルダ内**: `[ゲームフォルダ]/.gamestringer_backups/`
2. **エクスポートしたZIPパッケージ内**: `backup/`フォルダ

### バックアップの復元方法

#### 方法1: GameStringerアプリから
1. アプリの**バックアップ**セクションに移動
2. 復元するファイルを選択
3. **復元**をクリック

#### 方法2: 手動
1. `.gamestringer_backups/`でバックアップファイルを見つける
2. ファイルを元の場所にコピー
3. タイムスタンプを削除してリネーム

## 翻訳エディター

エディターでは手動で翻訳を編集できます。

### 階層構造

```
📁 ゲーム
├── 📁 Decarnation
│   ├── 📄 dialogues.csv (897文字列)
│   └── 📄 items.csv (123文字列)
└── 📁 別のゲーム
    └── 📄 texts.json (456文字列)
```

### 機能

- **検索**: テキストで文字列を検索
- **フィルター**: 未完了の翻訳、エラーありなどのみ表示
- **AI提案**: 個別の文字列に対して新しい翻訳をリクエスト
- **自動保存**: 変更は辞書に保存

---

## 辞書

辞書は各ゲームの翻訳を保存します。

### 動作方法

1. 各ゲームに個別の辞書があります
2. 翻訳は自動的に保存されます
3. 将来の翻訳を高速化するために再利用
4. 様々な形式でエクスポート可能（JSON、CSV、TMX）

### 辞書の場所

```
%APPDATA%/GameStringer/dictionaries/
├── 1672310_decarnation.json
├── 123456_別のゲーム.json
└── ...
```

## トラブルシューティング

### 翻訳が遅い

- **原因**: 文字列が多すぎる、またはプロバイダーが遅い
- **解決策**: バッチサイズを増やすか、Gemini（より高速）を使用

### APIキーエラー

- **原因**: APIキーが無効または期限切れ
- **解決策**: プロバイダーのウェブサイトでキーを確認

### パッチャーがインストールされない

- **原因**: アンチウイルスがBepInExをブロック
- **解決策**: ゲームフォルダの例外を追加

### ファイルが認識されない

- **原因**: サポートされていないファイル形式
- **解決策**: CSVまたはJSONに変換

### フォーマットエラーのある翻訳

- **原因**: AIが変数やタグを変更した
- **解決策**: 「Quality Checks」を有効にして自動検出
