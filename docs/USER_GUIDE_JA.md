## 概要

GameStringerはビデオゲームの自動・手動翻訳のための高度なシステムです。以下をサポート:

- **ゲームエンジン**: Unity、Unreal Engine、RPG Maker、Ren'Py、Godot、Telltale、Wolf RPG、Kirikiriなど
- **ファイル形式**: CSV、JSON、XML、PO/POT、YAML、TXT、SRT、VTT、ASS/SSAなど
- **AIプロバイダー**: Claude、Gemini、GPT、DeepSeek、Mistral、Groq、Ollama、**Qwen 3**、**NLLB-200**（18以上のプロバイダー）
- **言語**: 200以上の対応言語（NLLB-200使用時）
- **多言語UI**: IT、EN、ES、FR、DE、JA、ZH、KO、PT、RU、PL（11言語）
- **ゲームストア**: Steam、Epic Games、GOG、Origin、Battle.net、Ubisoft、itch.io、Amazon Games
- **新機能 v1.0.5**: Voice Clone Studio、VR Text Overlay、Quality Gates、Player Feedback
- **新機能 v1.0.6**: Qwen 3（アジア言語）、NLLB-200（200言語）、バグ修正
- **新機能 v1.0.7**: Community Hub、GitHub Discussions、ライセンス v1.1
- **新機能 v1.0.8**: アップデートダウンロード修正
- **新機能 v1.0.9**: アニメーションヘッダー、アップデート通知、UIポリッシュ

## 初回起動とプロファイル

### プロファイル作成

初回起動時、GameStringerはユーザープロファイルの作成を求めます：

1. **「プロファイル作成」をクリック**（初期画面）
2. **プロファイル名を入力**（例：「私の名前」）
3. **パスワードを設定**（最低6文字）
4. **「作成」をクリック**して確認

### ログイン

既存のプロファイルにアクセスするには：

1. **リストからプロファイルを選択**
2. **パスワードを入力**
3. **（オプション）**「パスワードを記憶」にチェックで自動ログイン
4. **「ログイン」をクリック**

### プロファイル管理

- **プロファイル切替**: 右上のプロファイルアイコンをクリック → 「プロファイル切替」
- **ログアウト**: プロファイルアイコンをクリック → 「ログアウト」
- **プロファイル設定**: 設定 → プロファイルへ

## ライブラリとゲーム詳細

### ライブラリ

ライブラリにはSteam、Epic Games、GOGなどのストアから同期された全ゲームが表示されます。

- **更新**: ゲームリストを再読み込み
- **共有**: ファミリーシェアリングゲームの表示/非表示
- **フィルター**: プラットフォーム、インストール状態、エンジンでフィルタリング

### ゲーム詳細ページ

ゲームをクリックすると**3:1**レイアウトの詳細ページが開きます：

#### メインカラム（75%）

- **スクリーンショットギャラリー**: クリック可能な最大12枚のスクリーンショット（ライトボックス）
- **クイック情報**: エンジン、ファイル数、インストールパス、DLC
- **ファイル/翻訳/パッチタブ**:
  - **ファイル**: 「Neural Translator」ボタン付きの翻訳可能ファイル
  - **翻訳**: このゲームのアクティブな翻訳
  - **パッチ**: Unity、Unreal、RPG Maker用パッチのインストール/削除

#### 右サイドバー（25%）

- **ゲーム情報**: 開発者、パブリッシャー、発売日、ジャンル、対応言語
- **アクション**: ゲームを翻訳、ファイルをスキャン
- **HowLongToBeat**: ゲームクリアの推定時間

#### 翻訳推奨

ページ下部でシステムがゲームを分析し、**最適な翻訳方法**を提案：

| 方法 | 使用タイミング |
|------|---------------|
| **Live Unity** | BepInEx + XUnity搭載のUnityゲーム |
| **File Translation** | ローカライズファイルが見つかった場合（JSON、CSVなど） |
| **OCR Overlay** | ファイルが見つからない場合、リアルタイム視覚翻訳 |

## Neural Translator Pro

### ファイルの翻訳方法

1. **Steamライブラリからゲームを選択**または手動でアップロード
2. **翻訳するファイルを読み込み**（ドラッグ＆ドロップまたは参照）
3. **オプションを設定**:
   - **ソース言語**: 元の言語（例：英語）
   - **ターゲット言語**: 翻訳先の言語（例：日本語）
   - **AIプロバイダー**: Claude（推奨）、Gemini、GPT
   - **APIキー**: 選択したプロバイダーのAPIキーを入力
4. **「翻訳開始」をクリック**して翻訳を開始
5. **進捗バーで進行状況を確認**

### 詳細オプション

| オプション | 説明 |
|----------|------|
| **Quality Checks** | 自動品質チェック（数字、フォーマットなど） |
| **Translation Memory** | 以前の翻訳を再利用して高速化 |
| **Batch Size** | 並列翻訳する文字列数（デフォルト: 10） |

### 推定コスト

システムは開始前にコスト見積もりを表示：

- **Claude**: 1Kトークンあたり〜$0.003
- **Gemini**: 1Kトークンあたり〜$0.0005（より安価）
- **GPT-4**: 1Kトークンあたり〜$0.01

## Translation Wizard

Translation Wizardはゲームファイルを自動翻訳するためのガイド付き手順です。

### Wizardの使い方

1. **Translatorに移動** → 「Translation Wizard」をクリック
2. **ライブラリからゲームを選択**またはパスを手動入力
3. **ファイルをスキャン**: Wizardが自動的に翻訳可能ファイルを検出
4. **翻訳するファイルを選択**（複数選択可能）
5. **オプションを設定**:
   - ソース言語とターゲット言語
   - AIプロバイダー
   - 品質オプション
6. **バッチ翻訳を開始**
7. **進捗バーで進行状況を確認**

### 自動検出フォーマット

| 拡張子 | タイプ |
|--------|--------|
| `.json` | JSONローカライズ |
| `.csv` | テキストテーブル |
| `.xml` | XML設定 |
| `.po/.pot` | Gettext（Linux標準） |
| `.txt` | プレーンテキスト |
| `.yaml` | YAML設定 |

## Translation Bridge

Translation BridgeはUnityゲームを**リアルタイム**でプレイ中に翻訳できます。

### 要件

- Unityゲーム（MonoまたはIL2CPP）
- BepInExインストール済み
- XUnity.AutoTranslatorプラグイン

### 設定方法

1. **メニューからTranslation Bridgeに移動**
2. **リストからUnityゲームを選択**
3. **BepInExをインストール**（未インストールの場合は自動）
4. **XUnity.AutoTranslatorを設定**:
   - ターゲット言語
   - 翻訳エンドポイント
5. **ゲームを起動** - 翻訳が自動的に表示されます

### 動作モード

- **ローカルキャッシュ**: 翻訳を再利用のために保存
- **ライブ翻訳**: 新しい文字列をその場で翻訳
- **フォールバック**: オフライン時はキャッシュのみ使用

## 公開API v1

> v1.0.4の新機能

### 利用可能なエンドポイント

| メソッド | エンドポイント | 説明 |
|---------|---------------|------|
| `POST` | `/api/v1/translate` | 単一文字列の翻訳 |

### リクエスト例

```json
POST /api/v1/translate
{
  "text": "Hello, world!",
  "source": "en",
  "target": "ja",
  "provider": "gemini"
}
```text

### レスポンス例

```json
{
  "translation": "こんにちは、世界！",
  "source": "en",
  "target": "ja",
  "provider": "gemini",
  "tokens": 12
}
```text

APIは自動ビルドパイプラインへのGameStringer統合に最適です。

## Quality Gates

> v1.0.5の新機能

翻訳の自動品質管理システム。

### 自動チェック

| チェック | 説明 |
|---------|------|
| **Placeholder** | {0}、{1}、%sなどを検証 |

| **長さ** | 翻訳が長すぎ/短すぎないか |
| **句読点** | 元のテキストとの一貫性 |

### 信頼度レベル

| レベル | スコア | 色 |
|--------|--------|-----|
| 🔴 クリティカル | < 40% | 赤 |
| 🟠 低 | 40-59% | オレンジ |
| 🟡 中 | 60-74% | 黄 |
| 🟢 高 | 75-89% | 緑 |
| 💚 完璧 | 90-100% | 濃緑 |

### 使い方 (5)

1. **メニューからQuality Gatesに移動**
2. **翻訳を読み込み**（JSON、CSV、または貼り付け）
3. **各文字列を自動分析**
4. **信頼度レベルでフィルタリング**
5. **JSONでレポートをエクスポート**

## トラブルシューティング

### 翻訳が遅い

- **原因**: 文字列が多すぎる、またはプロバイダーが遅い
- **解決策**: バッチサイズを増やすか、Gemini（より高速）を使用

### APIキーエラー

- **原因**: APIキーが無効または期限切れ
- **解決策**: プロバイダーのウェブサイトでキーを確認

### パッチャーがインストールされない

- **原因**: アンチウイルスがBepInExをブロック
- **解決策**: ゲームフォルダの例外を追加

### ファイルが認識されない

- **原因**: サポートされていないファイル形式
- **解決策**: CSVまたはJSONに変換

### フォーマットエラーのある翻訳

- **原因**: AIが変数やタグを変更した
- **解決策**: 「Quality Checks」を有効にして自動検出

## アダプティブMT

アダプティブMT（適応型機械翻訳）は、人間の修正から学習して翻訳品質を段階的に向上させるシステムです。

### 仕組み

1. **修正の保存**: AI翻訳を修正すると、ペア（原文 → 修正）が保存されます
2. **ファジー類似度**: トライグラム（Dice係数）+ 単語類似度（Jaccard）で関連する修正を検索
3. **Few-shot学習**: 現在のテキストに最も類似した修正がプロンプトに例として注入されます
4. **フィードバックループ**: 修正を多く保存するほど、将来の翻訳が向上します

### 機能

- **タグ自動検出**: tone_change、terminology、major_rewrite、length_change、punctuation、style
- **コンテキストブースト**: 同じゲーム（1.3x）、同じコンテンツタイプ（1.2x）、最近の修正を優先
- **承認**: 修正を検証済みとしてマークし信頼性を向上
- **インポート/エクスポート**: プロジェクト間で修正セットをエクスポート・インポート
- **統計**: 言語、ゲーム、タイプ、タグ、平均使用回数別の修正数

### 設定

| パラメータ | デフォルト | 説明 |
|---|---|---|
| **最大例数** | 5 | プロンプトあたりの最大few-shot例数 |
| **類似度しきい値** | 0.2 | 例を含めるための最小類似度 |
| **同じゲーム** | はい | 同じゲームの修正を優先 |
| **承認済みのみ** | いいえ | 承認済みの修正のみ使用 |

## バッチフォルダー翻訳

## 自動用語集

自動用語集は、LLMを使用してゲームテキストから用語を自動抽出し、ゲームごとの用語集に保存し、翻訳プロンプトに注入して用語の一貫性を保証します。

### 3段階システム

| レベル | アイコン | 動作 |
|---|---|---|
| **Locked** | 🔒 | 固定翻訳、AIによる変更なし |
| **Synced** | 🔄 | 優先翻訳、AIが代替を提案可能 |
| **Flexible** | 🔓 | 提案翻訳、AIが最適を選択 |

### 用語カテゴリ

👤 キャラクター、📍 場所、🎒 アイテム、⚔️ スキル、📜 クエスト、🖥️ UI、⚙️ システム、📚 ロア、🐉 クリーチャー、🏰 派閥、📌 その他

### 機能

- **自動抽出**: LLMでゲームテキストを分析しキーワードを抽出
- **デフォルト用語**: 一般的なゲーム用語を自動追加（HP、XP、NPCなど）
- **検索・フィルター**: テキストで検索、レベルやカテゴリでフィルタリング
- **プロンプト注入**: 用語が翻訳プロンプトに自動注入
- **翻訳禁止**: 翻訳すべきでない用語をマーク
- **大文字小文字区別**: 固有名詞用の大文字小文字オプション
- **インポート/エクスポート**: CSVまたはJSON形式で用語集をインポート・エクスポート
- **一貫性チェック**: 翻訳での用語の統一使用を検証
- **統計**: レベル、カテゴリ、ソース（自動/手動）別の用語数

### 設定

| パラメータ | デフォルト | 説明 |
|---|---|---|
| **有効** | はい | 自動用語集の有効/無効 |
| **最初のバッチで抽出** | はい | 最初の翻訳バッチから用語を抽出 |
| **抽出あたり最大用語数** | 20 | 1回あたりの最大抽出用語数 |
| **最小信頼度** | 50 | 最小信頼度閾値（0–100） |
| **プロンプトに注入** | はい | 翻訳プロンプトに用語を注入 |
| **プロンプト内最大用語数** | 30 | プロンプトあたりの最大用語数（コンテキストウィンドウ制限） |

### 使い方

1. サイドバーの**用語集**に移動
2. ゲーム、ソース言語、ターゲット言語を選択して新しい用語集を作成
3. 手動で用語を追加するか**「用語抽出」**をクリックしてAI抽出
4. 各用語のレベル（Locked/Synced/Flexible）とカテゴリを設定
5. 用語は翻訳プロンプトに自動注入
6. **「一貫性チェック」**を使用して用語の統一使用を確認
